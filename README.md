Tema 3 POO - Agentie de Turism continuare

Proiectul implementeaza un sistem de gestiune a unei agentii turistice. Codul este organizat in multiple clase distincte ce reprezinta componentele principale ale agentiei: turisti, servicii, precum si subclasele clasei serviciu, tranzactie, exceptii, mecanism de logger, precum si clasa catalog. Proiectul se foloseste de doua design pattern-uri, Observer si Iterator, ambele implementate prin clase template.
Clasa Turist modeleaza datele personale ale unui client precum numele, prenumele, varsta si numarul de telefon. Pe langa constructor, getteri si setteri, ce sunt prezenti intr-o multitudine din clase, aceasta clasa prezinta si metoda afiseazaTurist, ce printeaza datele clientului, precum si un operator<< folosit pentru afisare in stream-uri.
Clasa Serviciu este o clasa abstracta ce modeleaza datele unui concept generic de serviciu turistic, continand date despre numele serviciului, locatia, data plecarii si intoarcerii si costul. Aceasta este utilizata ca o baza peste care subclasele ei vor fi realizate.
Subclasa Cazare adauga ca date mic dejun inclus, acesta fiind folosind in calcularea costului final, acesta depinzand atat de micul dejun inclus cat si de varsta clientului.
Subclasa Excursie adauga ghid inclus si costul acestuia. Din nou, aceste date sunt folosite pentru calcului costului final, similar cu subclasa Cazare.
Subclasa Zbor adauga o taxa de bagaj precum si bagaj inclus. Structura este identica subclaselor.
Clasa Tranzactie modeleaza datele unei tranzactii, aceasta depinzand de clasele Turist si Serviciu, precum si un numar total de tranzactii. Clasa prezinta si metoda procesareTranzactie, pentru utilizarea static_cast-ului, ce afiseaza datele finale complete ale tranzactiei.
Clasa Exceptii este folosita prin mostenirea std::exception pentru a stoca mesaje de eroare. Aceasta utilizeaza subclase pentru tipurile de erori posibile: ExceptieDataInvalida, ExceptieCapacitate si ExceptieCost, fiecare cu mesajele lor distincte.
Cele doua design pattern-uri au fost implementate in major in clasa Catalog, precum si in main. Ambele se folosesc de clase template abstracte. Pattern-ul Iterator a fost primul implementat cu metodele first, next, current si isDone. Iteratorul este folosit pentru parcurgerea datelor unui catalog folosind vector. Observer a fost adaugat pentru incorporarea unui logger al proiectului. Acesta se foloseste de 3 subclase: LoggerTurist, LoggerServiciu si LoggerTranzactie fiecare fiind utilizate la adaugarea in catalog a tipurilor de date specifice.
Clasa Catalog este o clasa template ce stocheaza elemente ale claselor anterioare intr-un vector. Aceasta are ca date un titlu si o lista de observatori. Metodele sale principale sunt adauga, elimina, adaugaObservator, getIterator si afiseazaCatalog. Functia template afiseazaElemente este declarata ca friend si are acces asupra vectorului privat elemente, afisand elementele folosindu-se de operator<<. Este valida utilizarii numai pentru clasa Turist.
Functia main implementeaza un meniu interactiv folosind case-uri cu optiuni pentru creari de Turisti, Servicii, Tranzactii, adaugarea acestora in cataloagele lor specifice si afisarea lor, precum si iesirea din program. Pentru fiecare optiune este verificat si input-ul, afisand o eroare pentru input invalid.
