# AgentieTurism-Tema-3
Obiectivul principal al poriectului este de a modela interacțiunea complexă dintre entitățile de bază ale domeniului: clienții (Turist), serviciile oferite (clasificare ierarhică prin Serviciu, Cazare, Zbor, Excursie) și operațiunile comerciale (Tranzactie).
Clasa Serviciu reprezintă fundamentul ierarhiei polimorfice. Aceasta împiedică instanțierea directă a unui serviciu generic, forțând utilizatorul API-ului să folosească concepte concrete precum Cazare sau Zbor. 
Clasa Tranzactie gestionează resurse alocate dinamic pe heap prin intermediul membrului Serviciu* serviciul_ales. Codul implementează regula celor trei: Constructorul de copiere, Operatorul de atribuire si Destructorul. Implementarea operatorului de atribuire folosește idiomul Copy-and-Swap. Prin transmiterea parametrului other prin valoare, se creează automat o copie pe stivă folosind constructorul de copiere. Ulterior, funcția swap schimbă stările interne între obiectul curent și copie.
Șablonul Prototype este implementat prin metoda virtuală pură Serviciu* clone() const. În clasa Tranzactie, când se dorește salvarea serviciului ales, codul apelează serv->clone(). Fără acest șablon, copierea unui obiect polimorfic ar fi dus la fenomenul de Object Slicing, unde doar partea de bază din Serviciu ar fi fost copiată, pierzându-se atributele specifice din Cazare sau Zbor. Prin Prototype, responsabilitatea duplicării este delegată obiectului instanțiat la runtime, asigurând decuplarea totală a clasei Tranzactie de restul ierarhiei. 
Pentru a simula o autoritate centrală, clasa AgentieTurism a fost concepută ca un Singleton. Ea ascunde constructorii în zona privată și oferă acces prin metoda statică getInstance().
Fișierul Exceptii.h definește o structură ierarhică clară pentru managementul erorilor. Toate excepțiile specifice (ExceptieDataInvalida, ExceptieCapacitate, ExceptieCost) moștenesc din ExceptieAgentie, care la rândul ei extinde clasa standard std::exception.
Utilizarea corectă a specificatorilor virtual const char* what() const noexcept override asigură compatibilitatea totală cu mecanismele standard de prindere a erorilor din C++.
Din punct de vedere arhitectural, codul evoluează de la o structură rentabilă orientată pe obiecte către un sistem avansat, integrând bune practici din ingineria software: templates, mecanisme de tratare a excepțiilor, reflexie la timpul execuției și șabloane de proiectare.
